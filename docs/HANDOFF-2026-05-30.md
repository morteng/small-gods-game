## Two things to know

1. **The "unify the UI" goal was already ~half-done by pi.dev.** The nice
   token-based UI (`spirit-hud`, `main-menu`, `tutorial`, `settings-unified`,
   `rival-panel`, `minimap-panel`, `llm-display`, `llm-settings-new`) is the **live,
   wired UI** in `game.ts`. The legacy dark panels were the dead duplicates (deleted).
   Don't re-litigate this — it's settled and verified by direct reads of game.ts imports.

2. **`World` has TWO index layers.** `EntityRegistry` has its own spatial/kind/tile
   indexes AND `World` (world.ts) has its own separate `spatial`/`kindIdx`/`tagIdx`.
   `query()` uses World's. So when mutating `x/y/kind/tags`, use the new
   `World.updateEntity()` — it syncs both. Do NOT mutate entity position directly.

## Environment gotcha (will bite next session too)
Claude's temp dir (`/private/tmp/claude-501/...`) intermittently **corrupts/truncates
tool output** even with 18GB free disk — `Read` truncates small files, `Bash` echoes back
mangled commands (`bad math expression` errors). Cause: pi.dev runs in this repo
(`.pi/extensions/`), and there are live `claude --dangerously-skip-permissions` processes
that are pi.dev (NOT other editors — user confirmed safe). **Workarounds:** write output to
`/private/tmp/*.txt` then `Read` it; use tiny single-value Bash (`printf 'X=%s' "$(...)"`);
run `tsc`/`test` as background tasks and Read the `.output`. **When reads start degrading,
STOP editing** — don't exact-match Edit against a file you can't reliably read.
